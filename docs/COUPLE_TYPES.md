## 一、设计原则

### 1. 命名规则：三段式意象化

**格式**：`自然意象 + 关系动态描述`

- ❌「守卫者型」（单人性格，看不出关系）
- ✅「双塔守望型」（双方都是塔，互相守望）
- ✅「风筝与线型」（一方是风筝一方是线，动态关系）

**命名来源**：
- 自然意象：烟火、风筝、藤蔓、老井、火焰、星星
- 动物意象：猫、激光笔、双生蝴蝶
- 神话意象：凤凰、双生火焰、伊甸园
- 动态隐喻：灯塔与舟、渡人与舟

**原则**：每个名字都能脑补画面，一句话讲清关系模式。

### 2. 配对机制：必须双方都答题

```
A 答题 → 完全看不到结果 → 强制邀请
   ↓
A 分享测试链接（不是结果）→ B/C/D 扫码
   ↓
B/C/D 选择路径：
   路径 1：「我是 A 的伴侣」→ 答题 → 合并 A+B → 生成独立结果
   路径 2：「我不是 A 的伴侣」→ 答题 → 成为新发起人
   ↓
B 完成后：B 能看 A+B 结果 → B 主动分享结果卡片给 A
   ↓
A 点卡片 → 查看 A+B 结果（含双方头像昵称）
```

### 3. 性别触发机制

- 用户注册时选性别（男/女）
- 系统按双方性别组合触发不同类型库：
  - 男 + 女 → 异性恋类型库（30 常见 + 5 稀有）→ genderCombo: "male-female"
  - 男 + 男 → 男男专属类型库（15 种）→ genderCombo: "male-male"
  - 女 + 女 → 女女专属类型库（15 种）→ genderCombo: "female-female"

### 4. 三维度组合（底层匹配逻辑）

| 维度 | 来源 | 取值 |
|------|------|------|
| 依恋风格 | ECR-R 量表 | 安全 S / 焦虑 A / 回避 D / 混乱 F |
| 冲突模式 | Gottman 四骑士 + Sue Johnson 追-逃循环 | 追-逃 / 逃-追 / 追-追 / 逃-逃 / 修复型 |
| 关系阶段 | Sternberg 爱情三角 + Murstein 阶段 | 蜜月 / 磨合 / 稳定 / 危机 / 重生 |

### 5. code 命名规则

- 大写蛇形（UPPER_SNAKE_CASE）
- 异性恋类型：主题词（如 OLD_COUPLE / KITE_LINE）
- 男男专属：MM_ 前缀（如 MM_MIRROR / MM_LIGHTHOUSE）
- 女女专属：FF_ 前缀（如 FF_VINE / FF_DUAL_ANXIOUS）
- 稀有类型：R_ 前缀（如 R_TWIN_FLAME）

## 六、配对机制与结果查看（核心架构）

### 6.1 配对流程

```
A 答完 30 题
   ↓
A 看到全屏「邀请伴侣解锁结果」（完全无结果）
   ↓
A 点击「邀请 TA 测」→ 生成测试邀请链接（不含结果）
   ↓
A 分享到微信对话/朋友圈
   ↓
B/C/D 扫码进入小程序
   ↓
选择路径：
┌─ 路径 1：「我是 A 的伴侣」
│  ↓
│  B 答完 30 题
│  ↓
│  系统合并 A+B 答案 → 按性别组合触发类型库
│  ↓
│  B 能看到 A+B 完整结果（含双方头像昵称）
│  ↓
│  B 点击「分享结果给 TA」→ 生成结果卡片
│  ↓
│  A 点卡片 → 进入小程序 → 查看 A+B 结果
│
└─ 路径 2：「我不是 A 的伴侣」
   ↓
   B 答完 30 题 → B 成为新发起人 → B 邀请自己的伴侣
```

### 6.2 A 的首页状态

A 答完题后打开小程序：
- **不显示任何结果**
- 显示「邀请伴侣解锁」全屏引导页
- 底部小字：「已邀请 N 人 / X 人已完成」
- 点击已完成的人 → 显示「等 TA 分享结果给你」（不显示结果）

### 6.3 结果卡片元素

- 双方微信头像 + 昵称
- 类型名 + emoji
- 一句话标签
- 6 维度雷达图缩略
- 默契度分数
- 「测测你们是哪种」CTA + 小程序码

### 6.4 分享测试 vs 分享结果

| 分享类型 | 入口 | 卡片内容 | 跳转目标 |
|---------|------|---------|---------|
| 分享测试 | A 首页「邀请 TA 测」 | 「我测了我们的情侣类型，来测测你的」 | 测试开始页 |
| 分享结果 | B 结果页「分享给 TA」 | 双方头像 + 类型 + 雷达图 | 结果查看页 |

### 6.5 边缘情况

| 情况 | 处理 |
|------|------|
| A 邀请多人，多人完成 | A 收到多个结果卡片，每个独立 |
| B 完成但 A 没收到分享 | A 看不到结果，只能看到「X 已完成」 |
| 双方性别相同 | 触发同性专属类型库 |
| 双方答案严重矛盾 | 显示「你们的关系充满张力」+ 提示重测 |
| 30 天内重复测 | 提示「已测过，看上次结果」 |
| A 想重新测 | 30 天后解锁重测 |

## 八、和市面竞品的差异化总结

| 竞品 | 类型体系 | 我们的差异 |
|------|---------|-----------|
| Love Type 16 | MBTI 4 字母衍生 16 型 | 用真实心理学（依恋+Gottman），65 种更丰富 |
| 日本 Love Type 16 | 4 维度 16 型 | 我们加「关系阶段」维度，更立体 |
| 小红书测试小程序 | 玄学/星座 | 65 种有明确心理学锚点 |
| MBTI 配对 | 16×16=256 种配对 | 我们 65 种更精炼，每种都有故事 |
| Entie | 无类型体系 | 我们有完整裂变体系 |

**核心差异**：
1. **意象化命名**（三段式）而非字母代号
2. **同性恋专属类型**（男男 15 + 女女 15），市面无竞品
3. **双方配对机制**（必须双方都答题），不是单人测试
4. **结果分享 ≠ 测试分享**，避免认知混淆
